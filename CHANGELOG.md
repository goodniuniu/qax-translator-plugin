# 更新日志 (CHANGELOG)

本文档记录 QAX 智能划词翻译插件的版本更新历史。

---

## [1.3.0] - 2026-04-14

### 新增功能
- 🔘 **插件开关**：在 popup 设置页增加划词翻译开关，支持一键暂停/开启划词翻译功能
- 🅾️ **状态标识**：插件关闭时图标显示灰色 `OFF` badge，直观展示当前运行状态
- 👤 **作者信息**：在 popup 设置页底部新增作者说明（作者：GZC，联系电话：3710）

### 问题修复
- 🐛 **修复版本号不一致**：统一 `manifest.json` 和界面显示的版本号为 `1.3.0`

### 影响范围
- `popup.html` / `popup.js`：新增开关控件和作者信息展示
- `content.js`：根据开关状态控制划词翻译响应
- `background.js`：badge 状态同步和默认配置管理

---

## [1.2.1] - 2026-03-27

### Bug修复
- 🐛 **修复弹窗空指针错误**：解决快速划词或提前关闭弹窗时的 `Cannot read property 'style' of null` 错误
- 🔧 **定时器管理优化**：重构弹窗定时器管理逻辑，销毁弹窗时自动清除所有相关定时器

### 技术改进
- 添加统一的定时器管理对象 `popupTimers`，集中管理所有弹窗相关定时器
- 新增 `clearAllPopupTimers()` 函数，确保弹窗销毁时清理所有定时器
- 在所有异步回调（requestAnimationFrame、setTimeout）中添加空值检查
- 使用 `isConnected` 属性检测 DOM 元素是否仍在文档中
- 防止内存泄漏，移除弹窗时清理所有引用

### 影响范围
- `content.js`：弹窗管理逻辑重构

---

## [1.2.0] - 2026-03-27

### 新增功能
- 🎯 **场景选择**：新增翻译场景选择功能，支持"一般翻译"和"邮局海关审单"两种模式
- 📋 **场景切换**：用户可根据使用场景选择不同的翻译模式，提升翻译准确性

### 功能说明
- **一般翻译**：适用于日常翻译需求，提供标准的中文翻译
- **邮局海关审单**：针对邮局海关审单场景优化，提供更专业的翻译术语和格式

### 技术改进
- 优化了插件配置界面，新增场景选择选项
- 改进了翻译请求的参数传递机制

---

## [1.1.0] - 2026-03-26

### 新增功能
- 🖱️ **右键菜单翻译**：选中文本后右键点击「翻译选中文本」菜单项即可翻译
- 解决某些页面划词无法触发翻译的问题

### 优化改进
- 🔧 **兼容性优化**：移除 Promise 依赖，改用纯回调方式
- 兼容奇安信浏览器等不支持 Promise 的环境
- 使用 XMLHttpRequest 替代 fetch API
- 使用 document.execCommand('copy') 替代 navigator.clipboard.writeText()

### 配置更新
- 默认 API 地址更新为：`http://10.3.4.1:1025/v1`
- 默认模型名称更新为：`deepseekr1`

## [1.0.0] - 初始版本

## 版本号说明

本项目采用语义化版本控制 (Semantic Versioning)：

- **主版本号 (Major)**：不兼容的 API 修改
- **次版本号 (Minor)**：向下兼容的功能性新增
- **修订号 (Patch)**：向下兼容的问题修正

例如：`1.2.0` 表示主版本 1，次版本 2，修订号 0。

## 升级指南

### 从 v1.2.0 升级到 v1.2.1

1. 下载最新版本的插件包 `qax-translator-plugin-v1.2.1.zip`
2. 在浏览器中卸载旧版本插件
3. 按照安装说明重新安装 v1.2.1 版本
4. 无需额外配置，直接使用即可

### 从 v1.1.0 升级到 v1.2.0

1. 下载最新版本的插件包 `qax-translator-plugin-v1.2.0.zip`
2. 在浏览器中卸载旧版本插件
3. 按照安装说明重新安装 v1.2.0 版本
4. 在设置页面中选择适合的翻译场景

### 从 v1.0.0 升级到 v1.1.0

1. 下载最新版本的插件包 `qax-translator-plugin-v1.1.0.zip`
2. 在浏览器中卸载旧版本插件
3. 按照安装说明重新安装 v1.1.0 版本
4. 配置新的 API 地址和模型名称（如有变化）
