# 插件加载失败排查指南

## 问题描述
在谷歌浏览器上可以正常加载插件，但放到内部网络后无法加载，报错提示："清单文件缺失或不可读取"（无法加载清单）

---

## 可能的原因

### 1. 文件复制问题

#### 问题现象
- 文件复制到内网后可能损坏
- 某些文件可能没有正确复制
- 文件编码可能发生变化

#### 排查方法
1. 检查 `manifest.json` 文件是否存在
2. 检查文件大小是否与源文件一致
3. 使用文本编辑器打开 `manifest.json`，确认内容完整

#### 解决方案
- 重新复制所有文件
- 使用压缩包（zip）传输，然后在目标电脑解压
- 确保复制时包含所有必要的文件

---

### 2. 文件权限问题

#### 问题现象
- 内网电脑可能对文件访问权限有限制
- 文件可能被设置为只读或无访问权限

#### 排查方法
1. 右键点击 `manifest.json` 文件
2. 选择"属性" → "安全"
3. 检查当前用户是否有读取权限

#### 解决方案
- 确保当前用户对插件文件夹有完全控制权限
- 将插件文件夹放在用户目录下（如 `Documents` 或 `Desktop`）
- 避免放在系统目录或受限制的网络驱动器上

---

### 3. 文件编码问题

#### 问题现象
- `manifest.json` 文件可能包含 BOM（字节顺序标记）
- 某些文本编辑器会自动添加 BOM，导致 Chrome 无法正确读取

#### 排查方法
1. 使用支持查看十六进制的编辑器打开 `manifest.json`
2. 检查文件开头是否有 BOM 标记（EF BB BF）

#### 解决方案
- 使用 VS Code 或其他代码编辑器打开 `manifest.json`
- 确保文件编码为 UTF-8（无 BOM）
- 保存文件时选择"UTF-8"而非"UTF-8 with BOM"

---

### 4. 文件路径问题

#### 问题现象
- 插件文件夹路径可能包含特殊字符
- 路径可能过长
- 路径可能包含非 ASCII 字符

#### 排查方法
1. 检查插件文件夹的完整路径
2. 确认路径中不包含特殊字符（如 `&`, `%`, `#` 等）
3. 确认路径长度不超过 260 字符（Windows 限制）

#### 解决方案
- 将插件文件夹放在简单的路径下，如：
  - `C:\Users\你的用户名\Desktop\qax-translator-plugin`
  - `C:\qax-translator-plugin`
- 避免使用中文路径或特殊字符

---

### 5. 文件缺失问题

#### 问题现象
- 某些必要的文件可能缺失
- `manifest.json` 中引用的文件不存在

#### 排查方法
1. 确认以下文件都存在：
   - `manifest.json`
   - `background.js`
   - `content.js`
   - `popup.html`
   - `popup.js`
   - `styles.css`
   - `diagnose.html`
   - `diagnose.js`
   - `icons/icon16.png`
   - `icons/icon32.png`
   - `icons/icon48.png`
   - `icons/icon128.png`

2. 对照 `manifest.json` 中的引用，确认所有文件都存在

#### 解决方案
- 重新复制缺失的文件
- 使用 `build.sh` 脚本重新构建插件（如果可用）

---

### 6. 企业策略限制

#### 问题现象
- 内网环境可能有企业策略限制
- 可能禁止加载未签名的扩展程序
- 可能禁止从特定位置加载扩展程序

#### 排查方法
1. 检查 Chrome 的策略设置：
   - 在地址栏输入 `chrome://policy/`
   - 查看是否有 `ExtensionInstallAllowlist` 或 `ExtensionInstallBlocklist` 策略
   - 查看 `ExtensionInstallSources` 策略

#### 解决方案
- 联系 IT 管理员，请求添加插件到允许列表
- 或者请求调整企业策略，允许加载开发者模式扩展

---

### 7. Chrome 版本问题

#### 问题现象
- 内网电脑的 Chrome 版本可能过旧
- Manifest V3 需要 Chrome 88 或更高版本

#### 排查方法
1. 打开 Chrome，在地址栏输入 `chrome://version/`
2. 查看版本号
3. 确认版本 >= 88

#### 解决方案
- 更新 Chrome 到最新版本
- 如果无法更新，考虑降级到 Manifest V2（不推荐）

---

## 正确的加载步骤

### 方法 1：加载已解压的扩展程序

1. 打开 Chrome
2. 在地址栏输入 `chrome://extensions/`
3. 打开右上角的"开发者模式"开关
4. 点击"加载已解压的扩展程序"
5. 选择包含 `manifest.json` 的文件夹
6. 确认插件出现在扩展列表中

### 方法 2：使用打包的扩展程序

1. 在开发机上打包插件：
   - 打开 `chrome://extensions/`
   - 开启"开发者模式"
   - 点击"打包扩展程序"
   - 选择插件根目录
   - 生成 `.crx` 文件

2. 将 `.crx` 文件复制到内网电脑
3. 拖拽 `.crx` 文件到 Chrome 扩展页面
4. 确认安装

## 快速检查清单

在尝试加载插件前，请确认：

- [ ] `manifest.json` 文件存在且格式正确
- [ ] 所有引用的文件都存在
- [ ] 文件编码为 UTF-8（无 BOM）
- [ ] 文件路径简单，无特殊字符
- [ ] 文件权限正确，可读取
- [ ] Chrome 版本 >= 88
- [ ] 已开启"开发者模式"
- [ ] 选择的是包含 `manifest.json` 的文件夹，而不是 `manifest.json` 文件本身

---

## 常见错误及解决

### 错误 1："清单文件缺失或不可读取"

**原因**：
- 选择了错误的文件或文件夹
- `manifest.json` 文件损坏
- 文件权限问题

**解决**：
- 确保选择的是包含 `manifest.json` 的文件夹
- 检查 `manifest.json` 文件是否可读
- 重新复制文件

### 错误 2："无效的清单文件"

**原因**：
- `manifest.json` 格式错误
- JSON 语法错误

**解决**：
- 使用 JSON 验证工具检查 `manifest.json`
- 确保所有逗号、引号都正确

### 错误 3："扩展程序无法加载"

**原因**：
- 缺少必要的文件
- 文件路径问题

**解决**：
- 检查所有引用的文件是否存在
- 确保文件路径正确

## 推荐的分发方式

### 方式 1：使用压缩包

1. 在开发机上，将插件文件夹压缩为 `.zip` 文件
2. 将 `.zip` 文件传输到内网电脑
3. 在内网电脑解压
4. 按照正确步骤加载插件

### 方式 2：使用企业策略（推荐）

如果内网环境有企业策略管理，可以：

1. 将插件打包为 `.crx` 文件
2. 将 `.crx` 文件上传到内网服务器
3. 通过企业策略自动分发插件

参考 `deploy/` 目录下的企业部署指南。

---

## 如果问题仍然存在

如果按照以上步骤仍然无法加载插件，请：

1. 记录完整的错误信息
2. 检查 Chrome 控制台（F12）是否有错误
3. 尝试在另一台内网电脑上加载，确认是否是单机问题
4. 联系 IT 管理员确认是否有企业策略限制
