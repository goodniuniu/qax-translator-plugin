❓ 常见问题
ℹ️ 服务信息（根据管理员反馈）

• 部署方式: vLLM

• 可用模型: MiniMax/MiniMax-M2.5

• API 地址: http://10.3.4.21:8000/v1

HTTP 404 - 接口不存在

浏览器访问 http://10.3.4.21:8000/v1 返回 {"detail":"Not Found"} 是正常的。

正确的 API 地址应该是 http://10.3.4.21:8000/v1/chat/completions

请在设置页面确认 API 地址以 /v1 结尾，不要包含 /chat/completions

连接超时

1. 检查网络是否可以访问 10.3.4.21:8000

2. 检查后端服务是否已启动

3. 检查防火墙是否放行端口

模型相关（重要）

• 推荐做法：将模型名称留空，让后端自动选择默认模型

• 如果需要指定模型，请使用"获取模型列表"功能查看可用模型

• 模型名称必须准确，区分大小写

• 如果遇到 "Model Not Exist" 错误，请清空模型名称或使用正确的模型名

API Key 设置（重要）

• 内部部署服务（如 10.3.4.21）：通常不需要 API Key，可以留空

• 互联网大模型服务（如 OpenAI、智谱 AI 等）：必须设置正确的 API Key

• API Key 会在设置页面配置，此处仅供查看

• 如果遇到 HTTP 401 错误，请检查 API Key 是否正确

HTTP 401 错误处理

• 错误含义：未授权，需要提供正确的 API Key

• 可能原因：

1. 使用了互联网服务但未设置 API Key

2. API Key 填写错误或已过期

3. 内部服务配置了认证但未提供 Key

• 解决方案：根据服务类型正确配置 API Key

解析错误处理

• 如果看到 "<think>" 标签，说明模型输出了思考过程

• 插件已自动处理此情况，请尝试再次测试

• 如仍有问题，请联系管理员调整模型提示词配置
